# Synchronised Sliders

What is this? Let's give a background story. You're tasked with creating a timeline component. The timeline component is a slider that allows you traverse through time. Time here can be a number of years, 10, 5, 100, 2, or 4.5 years. Here's the catch, every step of the time is in a hour in a year. This with 24 hours in day and 365 days in a year, there a 8,760 hours to possibly seek. Now we want to be able to view every month in that year. Obviously if we are able to view at an hourly resolution. We've covered monthly we just to aggregrate up. Another catch, I want a slider for each month in a year and so I can easily jump between months within. I also want a slider for each hour incase I want to be precise. 

Of course, there multiple other features or constraints we can add to that and they sufficiently make it complex. For example, every update on the slider triggers a render of a separate component or we should be able to lock specific overs and months while seeking. However, the base scenario here is is complex enough. So let's take stab at this.

Consider we have slider component where we can render each 8,760 hours of a year for 10 years. I mean I don't know the kind of the screen you'd want to have this but for the sake of fun let's play.

# Constraints

- React renders at 60 fps and at 50 fps a user is then able to notice a lag in interactivity and rendering. We don't want that. So what do we do?

- Each slider can be interacted with and other sliders should respond accordingly.
- Only manage a singular variable for the state. You can use other variables to track the different sliders but only one for the value
